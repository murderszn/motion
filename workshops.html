01 · context engineering

How agents actually work.

Every mechanism is a way to put the right knowledge in front of Claude at the right time. The core question is when each one enters the context window — because context is finite, and what's loaded shapes every response.

Persistent

CLAUDE.md

Claude Code's memory file. Read automatically at session start and kept in context for the entire conversation. The place for things you'd otherwise repeat in every prompt — commands, conventions, architecture. Everything here costs tokens on every turn, so keep it tight and high-value.

Cross-vendor

AGENTS.md

A vendor-neutral standard: a plain-markdown instructions file that many different coding agents agree to read. The goal is one file your whole toolchain shares. Claude Code bridges it in with a one-line import: @AGENTS.md

On demand

Agent Skills

A folder with a SKILL.md file that packages expertise for a specific task type. The defining feature is progressive disclosure: only a name and one-line description sit in context. The full instructions load only when Claude judges the skill relevant. Cheap to have many.

Isolated

Subagents

A separate Claude with its own context window and tools. It does the job and reports back a summary without polluting your main thread. Use for big, isolatable jobs — heavy search, log parsing, or work needing its own tools. A skill can be configured to run-in-subagent.

  or acroform data.

files like reference.md — they load only when needed.

- Font: Inter (body), Inter Tight (headings)

The description is the most important line — it's the only thing Claude sees at rest. Write it in terms of when to use this, not just what it is.

02 · the agentic loop

Plan, execute, verify.

A coding agent is a loop: the model reads your context, plans a change, edits files, runs tools to check its work, and iterates until the task is done. Understanding this loop is the key to using agents effectively.

1. Context Load

CLAUDE.md, AGENTS.md, and relevant skills are loaded. The agent reads the codebase structure and understands the project.

2. Plan

The agent outlines what it will change and why. Good plans reference existing patterns in the codebase, not invent new ones.

3. Execute

File edits, terminal commands, API calls. The agent uses tool calls to modify the codebase — reads, writes, compiles, tests.

4. Verify

Run tests, check types, lint. The agent reads the output and decides if the change is correct or needs another iteration.

5. Iterate

If verification fails, the agent adjusts and re-runs. This loop continues until the task is complete or the agent asks for help.

6. Report

The agent summarizes what it did, what changed, and any decisions it made. For subagents, this summary is all you get back.

Why this matters

Most failed agent interactions come from poor context, not poor models. If the agent can't see your conventions, your test runner, or your architecture — it will guess. Context engineering is the skill that makes everything else work.

CLAUDE.md — keep it lean
# Project: Acme API

See @docs/architecture.md for the system overview.
Coding standards live in @docs/conventions.md.

## Commands
- build: npm run build
- test:  npm test -- --watch=false
- lint:  npm run lint

## Conventions
- language: TypeScript
- style: Functional, no classes
- tests: Vitest, co-located with source
- imports: Absolute from @/
Always true for this repo?
Commands, conventions, architecture that apply to every task → CLAUDE.md.
Shared across tools?
One file for Cursor, Codex, and Claude → AGENTS.md, imported from CLAUDE.md.
Repeatable specialized task?
A workflow that only matters sometimes, possibly with scripts → package a Skill.
Big isolatable job?
Heavy search, log parsing, or work needing its own tools → delegate to a Subagent.

04 · full stack workflow

Wiring it all together.

The real power is in the composition: an agent that reads your codebase, writes code, runs your CLI tools, deploys to the cloud, and verifies the result — all from one prompt.

Step 01

Context Setup

Write CLAUDE.md with your project's commands, conventions, and architecture. Import AGENTS.md for cross-tool compatibility. Add skills for repeated workflows.

Step 02

Agent-Assisted Development

Describe the feature in natural language. The agent plans, writes code, runs tests, and iterates. You review at each checkpoint — the human owns every claim.

Step 03

CLI-Driven Operations

Use gh to create PRs, databricks CLI to run queries, supabase CLI for migrations, gcloud to deploy. The agent can invoke these tools directly.

Step 04

Deploy & Verify

Push to GitHub, CI runs automatically, Cloud Run deploys, Supabase migrations apply. The agent verifies the deployment and reports back.

05 · 3d graphics & shaders

Shaping pixels with the GPU.

Modern visual interfaces aren't static images—they are live-rendered pipelines. Harnessing WebGL, fragment shaders, and Three.js opens up high-performance procedural graphics and hardware post-processing filters directly in the browser.

Procedural

Textureless Perlin Noise

Generating organic textures (clouds, liquids, flow fields) on-the-fly without loading image assets. Uses mathematical lattices and quintic Hermite interpolation curves to map coordinates to smooth random values directly on the GPU.

Pure Shader

Signed Distance Fields (SDFs)

Defining geometries mathematically inside the fragment shader. Instead of standard polygons, shapes are calculated using vector distance functions, providing infinite rendering resolution independent of scale.

Post-Process

Depth of Field & CoC

Simulating camera lens optics by separating the color scene and linear view-space depth maps. The fragment shader computes a per-pixel Circle of Confusion (CoC) based on focal distance, applying variable-width Gaussian blurs.

Optimization

Separable Blur Convolutions

Reducing GPU performance bottlenecks. A 2D Gaussian blur is split into consecutive 1D horizontal and vertical blur passes, scaling down texture fetches from N² samples to 2N samples for high framerates.

Circle of Confusion Shader Math

postprocess.glsl — depth-based lens blur
// Renders depth map and calculates lens blur coefficient
uniform sampler2D u_depthTexture;
uniform float u_focusDistance;
uniform float u_apertureSize;

void main() {
    float depth = texture2D(u_depthTexture, vUv).r;
    
    // Circle of Confusion (CoC) calculation (NVIDIA GPU Gems 3, Ch 28)
    float coc = abs(depth - u_focusDistance) * u_apertureSize;
    
    // Clamp blur radius and apply variable-width kernel offsets...
    gl_FragColor = vec4(applyBlur(vUv, coc), 1.0);
}

By combining depth pass buffers with shader mathematical filters, agents can build premium responsive graphic effects like glassmorphism overlays and camera lens focuses.
